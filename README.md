# LINE 公式アカウントへメッセージを送る（Python）

このフォルダのプログラムは、**LINE 公式アカウント**と**Messaging API（メッセージングAPI）**を使って、指定した相手にテキストメッセージを送ります。

## 1. 初期設定（必ず最初にここから）

ここでは「パソコンにまだ Python がない人」「コマンド画面をほとんど使ったことがない人」でも進められるように書きます。**上から順に**実施してください。

### 1-1. Python をインストールする

1. ブラウザで次の公式ページを開きます。  
   https://www.python.org/downloads/
2. 大きな黄色いボタン（**Download Python 3.x.x**）を押して、インストーラーを保存します。
3. 保存したファイルを開き、インストールを開始します。
4. **重要:** 最初の画面に **「Add python.exe to PATH」**（または「Add Python to PATH」）というチェックがあれば、**必ずオン（チェックを入れる）**にしてから「Install Now」を選びます。  
   これを忘れると、後で「python が認識されない」エラーになりやすいです。
5. インストールが終わったら、パソコンを**一度再起動**すると安心です（必須ではありませんがおすすめです）。

### 1-2. インストールできたか確認する

1. Windows の場合、**スタートメニュー**から「**cmd**」または「**PowerShell**」を開きます。
2. 次のコマンドを入力して Enter を押します。

```text
python --version
```

3. `Python 3.x.x` のように表示されれば成功です。  
   もし `python` で失敗する場合は、次も試してください。

```text
py --version
```

4. こちらでバージョンが表示されたら、以降のコマンドでは `python` の代わりに `py` を使ってください（例: `py -m venv .venv`）。

### 1-3. このフォルダに移動する

コマンド画面で、この README と同じフォルダに移動します。

1. エクスプローラーで、このプロジェクトのフォルダ（`LINE` フォルダ）を開きます。
2. アドレスバーをクリックしてパスが青く選ばれたら、`Ctrl + C` でコピーします。
3. コマンド画面で次のように入力します（`貼り付けたパス` の部分に、コピーしたパスを貼り付けます）。

```text
cd 貼り付けたパス
```

例（ユーザー名や場所は人によって違います）:

```text
cd C:\Users\あなたの名前\LINE
```

### 1-4. 仮想環境を作る（おすすめ）

他のプログラムとライブラリのバージョンが混ざらないよう、このフォルダ専用の「仮想環境」を作ります。失敗しても消せばよいので、初めてでもここはやっておく価値があります。

```text
python -m venv .venv
```

`python` が使えない場合:

```text
py -m venv .venv
```

### 1-5. 仮想環境を有効にする

**PowerShell** の場合（このドキュメントの想定）:

```text
.\.venv\Scripts\Activate.ps1
```

初めてのとき、**スクリプトの実行が禁止**と出ることがあります。その場合は、**一度だけ**管理者ではない通常の PowerShell を開き、次を実行してから、もう一度 `Activate.ps1` を試してください。

```text
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**コマンドプロンプト（cmd）**の場合:

```text
.venv\Scripts\activate.bat
```

成功すると、行の先頭に `(.venv)` のように表示されることが多いです。

### 1-6. 必要なライブラリを入れる

仮想環境を有効にした状態で、次を実行します。

```text
python -m pip install -r requirements.txt
```

`python` が使えない場合:

```text
py -m pip install -r requirements.txt
```

### 1-7. 設定ファイル（.env）を作る

1. このフォルダにある **`.env.example`** というファイルをコピーし、名前を **`.env`** にします。  
   - コピー方法の例: エクスプローラーで `.env.example` を右クリック → コピー → 貼り付け → 名前を `.env` に変更。
2. `.env` をメモ帳などで開き、後述の手順で手に入れた **チャネルアクセストークン**と**ユーザーID**を貼り付けて保存します（中身の詳細は「[5. 送信に必要な値の準備](#5-送信に必要な値の準備)」）。

**注意:** `.env` には秘密の文字列が入ります。**他人に見せないでください。** また、**写真付きSNSやブログに貼らないでください。**

## 2. 用語の簡単な説明

読み進める前に、後で出てくる言葉を短く整理します。

| 言葉 | 意味 |
|------|------|
| **LINE 公式アカウント** | 店舗やサービスが運営する「公式の LINE の窓口」。友だちとして追加できる。 |
| **LINE Developers** | LINE が提供する開発者向けのサイト。プログラム連携の設定をする場所。 |
| **Messaging API** | プログラムから LINE にメッセージを送ったり受けたりするためのしくみ。 |
| **チャネル（Channel）** | LINE Developers 上の「API を結ぶ窓口」の単位。ここにトークンが紐づく。 |
| **チャネルアクセストークン** | プログラムが「この公式アカウントとして送る権限がある」ことを示す秘密の文字列。 |
| **チャネルシークレット** | Webhook（後述）の正当性を確認するために使う秘密。この送信だけなら必須ではないことが多い。 |
| **ユーザーID（送信先）** | メッセージを届ける相手ごとの固有のID。**電話番号や LINE の表示名ではありません。** |

この README が前提としているのは、送信される側が **あなたの LINE 公式アカウントを友だちとして追加していること**です。

## 3. 公式LINE（LINE公式アカウント）の作成手順

「公式LINE」とは、一般に **LINE 公式アカウント**のことです。次の流れで作成します。

### 3-1. LINE 公式アカウントを開く

1. ブラウザで次を開きます。  
   https://www.linebiz.com/jp/service/line-official-account/
2. 画面上の案内に従い、「無料で始める」や「アカウントを作成」など、作成開始ボタンを押します。

### 3-2. アカウントタイプの確認（概要）

- LINE は機能によりタイプ（無料・プレミアム等）の差があります。Messaging API を安定して使う場合は、**LINE の案内に従い、必要に応じてプランを確認**してください。  
- 料金や契約内容は時期で変わるため、**必ず公式の説明画面**を読んでから進めてください。

### 3-3. 基本情報の登録

1. LINE の案内に従い、**アカウント名**、**業種**、**連絡用のメールアドレス**などを入力します。
2. 必要に応じて、**本人確認**や**利用規約への同意**を行います（画面の指示に従ってください）。

### 3-4. 管理画面に入れることを確認する

1. **LINE 公式アカウント マネージャー**（管理画面）にログインできる状態にします。  
2. ここで、あとから友だち追加用のQRコードやURLを確認できるようになります。

### 3-5. スマートフォンで友だち追加（テスト用）

**自分のスマートフォン**で、今作った公式アカウントを**友だち追加**しておきます。  
メッセージ送信のテストは、多くの場合「自分の LINE に届くか」で確認するのが簡単です。

## 4. LINE Developers で Messaging API を有効にする

### 4-1. LINE Developers にログインする

1. ブラウザで https://developers.line.biz/ を開きます。
2. 右上などからログインします（LINE アカウントでログインします）。

### 4-2. プロバイダーを作る（初めての場合）

1. コンソール（管理画面）で **Provider（プロバイダー）**を作成します。  
2. 名前は分かりやすい任意の名前で構いません（例: 個人名、屋号など）。

### 4-3. チャネルを作成する（Messaging API）

1. **Create a new channel**（新しいチャネルの作成）のようなメニューから、**Messaging API** のチャネルを選びます。
2. 表示名や説明を入力し、利用規約に同意して作成します。

### 4-4. 既存の「LINE 公式アカウント」と紐づける

作成途中または作成後の画面で、**先ほど作った LINE 公式アカウント**とこの Messaging API チャネルを**連携（リンク）**する流れになります。  
画面の案内に従ってください（表示は更新されることがあります）。

### 4-5. チャネルアクセストークン（長期）を発行する

1. 作成した **Messaging API** チャネルの設定画面を開きます。
2. **Messaging API settings** などの項目にある **Channel access token**（チャネルアクセストークン）で **Issue**（発行）を押します。  
3. 表示された長い文字列をコピーし、`.env` の `LINE_CHANNEL_ACCESS_TOKEN=` の右側に貼り付けます。  
4. **この文字列は秘密です。** 誰にも見せないでください。

### 4-6. Webhook について（送信だけなら最低限）

メッセージを「送るだけ」の場合でも、LINE の設定画面には **Webhook URL** の項目があります。  
まずは送信テスト優先で構いませんが、ユーザーIDを取得する簡単な方法として後で Webhook を使う説明もします（「[5-3](#5-3-送信先のユーザーidを入手する)」）。

## 5. 送信に必要な値の準備

### 5-1. チャネルアクセストークン

「[4-5. チャネルアクセストークン（長期）を発行する](#4-5-チャネルアクセストークン長期を発行する)」で取得した文字列を `.env` に書きます。

```env
LINE_CHANNEL_ACCESS_TOKEN=ここに貼り付け
```

### 5-2. 送信先のユーザーIDとは

LINE はセキュリティのため、プログラムから送る宛先を **「ユーザーID」** で指定します。  
これは **U で始まる英数字の文字列**であることが多く、**あなたが決められるものではありません**。

### 5-3. 送信先のユーザーIDを入手する

初心者向けに現実的な方法をいくつか挙げます。**どれか一つ**できれば十分です。

**方法A（おすすめ: 自分宛てに送ってテストする）**  
まず **自分のスマートフォン**から、公式アカウントに**友だち追加**します。  
次に、次のいずれかで **自分のユーザーID** を取得します。

- **A-1: Webhook を一時的に使う（プログラムが少し増える）**  
  ユーザーが公式アカウントにメッセージを送ると、LINE からあなたのサーバーに通知（Webhook）が届き、その通知の中に **userId** が含まれます。  
  そのため、インターネットに公開できるURL（初心者向けには **ngrok** などのトンネルツール）が必要になることがあります。  
  今回のフォルダは「送信だけ」に絞っているため、Webhook 用のサーバーは同梱していません。必要なら別途、Webhook 受信のサンプルを追加で用意する方法があります。

- **A-2: 既存の仕組みで userId を確認できる場合**  
  すでにLINE連携のシステムを持っている場合、管理画面やログに userId が出ることがあります。

**方法B: 開発用の簡易ボットで userId を表示する**  
インターネット上のチュートリアルに「メッセージを受け取ったら userId を返す」サンプルがあり、それを一時的に動かして ID を控える、というやり方もよく使われます。

**重要:** 取得した **ユーザーID も秘密情報として扱う**のが安全です（悪用されると、あなたの公式アカウントからその人にメッセージを送られる可能性があるため）。

`.env` には次のように書きます。

```env
LINE_USER_ID=Uで始まる文字列
```

### 5-4. 送る文章（任意）

省略した場合、プログラムはデフォルトの短い文を送ります。変えたい場合は `.env` に次を追加します。

```env
LINE_MESSAGE=こんにちは。テスト送信です。
```

## 6. このプログラムの実行方法

### 6-1. 事前確認

- 仮想環境を有効にする（「[1-5. 仮想環境を有効にする](#1-5-仮想環境を有効にする)」）
- `.env` に `LINE_CHANNEL_ACCESS_TOKEN` と `LINE_USER_ID` がある

### 6-2. 実行コマンド（基本）

```text
python send_line_message.py
```

`python` が使えない場合:

```text
py send_line_message.py
```

成功すると `送信に成功しました。` と表示されます。

### 6-3. 文章だけコマンドで上書きする

```text
python send_line_message.py -m "本日のお知らせです。"
```

### 6-4. 送信先だけコマンドで上書きする

```text
python send_line_message.py -u "Uxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

## 7. よくあるトラブル

### 7-1. `401 Unauthorized` や「Invalid access token」

- **チャネルアクセストークン**のコピー漏れ、余分な空白、古いトークンを使っていないか確認します。
- `.env` を保存し忘れていないか確認します。

### 7-2. `400 Bad Request` や「Invalid user id」

- ユーザーIDが **正しい相手のもの**か確認します（別のチャネルで作ったIDと混同しやすいです）。
- 相手が **その公式アカウントを友だち追加しているか**確認します。

### 7-3. 送信できない（ブロック）

相手が公式アカウントを**ブロック**していると届きません。友だち追加とブロック解除を確認してください。

### 7-4. `python` が見つからない

- Python 再インストール時に **PATH 追加**を忘れていないか確認します。
- `py` コマンドで代用します。
